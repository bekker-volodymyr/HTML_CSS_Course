Блокова модель

Блокова модель (box model) - набір властивостей блокових елементів, що впливають на розмір елементу.

До блокової моделі входять:

  • внутрішні відступи (padding)
  • зовнішні відступи (margin)
  • товшина рамки (border-width)
  • висота елементу (height)
  • ширина елементу (width)

Блокова модель відштовхується від вмісту елементу (ширина та висота вмісту визначається властивостями width/height). Інші властивості (border-width, padding, margin) поширюються на всі боки контенту.

Ширина блокового елементу

Для задання ширини використовується властивість width.

Властивість width підтримує значення auto. Це значення за замовченням. В такому разі блоковий елемент займає всю доступну ширину батьківського елементу.

Також для width можна назначити значення в px, em, vw, cm, mm та інших одиницях або у %. Коли ширина задається у відсотках, її підрахунок однаково виконується браузером в пікселях відносно розміру батьківського елементу.

Для контролю ширини також можна задати властивості max-width/min-width. max-width обмежує максимальний простір, що може зайняти елемент по ширині, а min-width не дозволяє елементу мати ширину менше за вказану.

Обидві властивості можуть бути вказані наступними значеннями:

  • max-width: розмір в px, pt, em, in, % та ін. одиницях | none | inherit;
  • min-width: розмір в px, pt, em, in, % та ін. одиницях | inherit.

За замовченням значення max-width встановлено none. Для min-width стандартне значення 0.

Приклад

Висота блокового елементу

Для висоти елементу, що визначається властивістю height, за замовчуванням задано значення auto, яке враховує висоту будь-якого блоку, що визначається в залежності від висоти його вмісту (контенту).

Частіше за все такий підхід змінювати не потрібно, тому що кількість текстів в елементі визначається замовником і, крім того, з часом може змінюватись, особливо в CMS — системах керування контентом. Тому жорстко обмежувати його не потрібно. Тим не менш, в CSS передбачені 3 властивості, що дозволяють задати висоту елементу наявно. Це властивості height, min-height і max-height.

Доступні значення:

  • height: розмір в px, pt, em, in, % та ін. одиницях | auto | inherit;
  • max-height: розмір в px, pt, em, in, % та ін. одиницях| none | inherit;
  • min-height: розмір в px, pt, em, in, % та ін. одиницях | inherit.

Для min-height значення за замовчуванням 0, для max-height — none.

Якщо значення властивості height буде менше, ніж його вміст, то вміст виходитиме за межі свого контейнера і наповзати на наступний текст.

Приклад

Властивість box-sizing

Як було зазначено раніше, властивості width та height задають лише ширину та висоту для області контенту, а відступи та рамка додаються до цих розмірів.

В результаті розміри елементу відрізняються від від очікуваних.

Ширина буде дорівнювати width + padding-left + padding-right + margin-left + margin-right + 2 x border-width.

Висота буде дорівнювати height + padding-top + padding-bottom + margin-top + margin-bottom + 2 x border-width.

Така ситуація може призвезти до труднощів та помилок. Наприклад при використанні функції calc() для центрального фіксованого позиціонування.

Для вирішення цієї проблеми існує властивість box-sizing. Потрібно задати цій властивості значення border-box. 

div {
    width: 500px;
    height: 350px;
    margin: 50px;
    padding: 10px;
    border-radius: 4px;
}  

div {
    width: 500px;
    height: 350px;
    margin: 50px;
    padding: 10px;
    border-radius: 4px;
    box-sizing: border-box;
}

Властивість overflow

Ця властивість визначає, як вестиме себе контент блокового елементу, якщо він повністю не поміщається в області, яку задають ширина і висота елементу.

Можливі значення:

  • visible
  • auto
  • hidden
  • scroll
  • inherit

Значення visible за замовчуванням показує весь вміст елементу, навіть, якщо він виходить за межі блоку.

Значення auto додає смугу (або 2 смуги) прокручування, коли вони необхідні.

Інструменти розробника (DevTools)

В процесі верстки сторінки або окремих її блоків у вас, напевно, будуть з'являтись складнощі, пов'язані з тим, що деякі CSS-властивості, які ви використовуєте, застосовуються не так, як ви очікували або не застосовуються зовсім.

Подивитись значення різних властивостей елементів і не тільки, можна, використовуючи Інструменти розробника, або Інспектор властивостей, який наразі є в кожному відомому браузері, і навіть викликається однаковим поєднанням клавіш: F12 або Ctrl+Shift+I.

Також ви можете відкрити Інспектор властивостей, обравши з контекстного меню пункт «Подивитись код елементу», «Подивитись код» або «Досліджувати елемент» по правому кліку на будь-якому елементі на HTML-сторінці.

Інструменти розробника зручно використовувати для дослідження наявних елементів та їх стилів.

Окрім того ви можете експерементувати зі значеннями CSS властивостей прямо в бравзері і бачити результат моментально. Наприклад ви можете підбирати кольори:

Плавуючі елементи: властивість float

Властивість float визначає, з якої сторони буде розташовано елемент (справа або зліва) від обтікаючого його тексту.

З боку CSS, властивість float може набувати таких значень:

  • float: left | right | none | inherit

Значення left вирівнює елемент зліва від обтікаючого його тексту, right вирівнює елемент справа від тексту, none — обтікання не застосовується до елементу (є значенням цієї властивості за замовчуванням), inherit — значення успадковується від батьківського елементу.

float: left;

Скасування float

Використання clear

У ряді випадків float-елемент може займати більше простору по висоті, ніж це необхідно відповідно до макету дизайнера або за логікою розташування текстових елементів. Весь текст або інші блоки, які в HTML-розмітці слідують за спливаючим елементом, будуть «обтікати» його зліва або справа, ламаючи зовнішній вигляд сторінки.

У цих випадках слід використовувати CSS-властивість clear, яка має наступні значення:

  • clear: left | right | both | none | inherit

Значення none є значенням за замовчуванням і жодним чином не впливає на обтікання елементів. Значення left застосовується для відміни обтікання елементів, що мають властивість float: left, right — для елементів із властивістю float: right, а значення both відміняє обтікання з обох боків, тому використовується в тих випадках, коли невідомо, яке було значення у властивості float, або обтікання застосовувалось із двох боків.

Приклад

Використання overflow

Другим способом відміни обтікання є використання властивості overflow із значеннями hidden або auto для батьківського блока-контейнера.

Справа в тому, що при розташуванні всередині якогось блокового елемента декількох спливаючих блоків, висота батьківського елемента «схлопується», тобто перетворюється в 0. При необхідності задати цьому батьківському елементу фоновий колір або фонове зображення, ви неминуче зіткнетеся з тим, що його висота дорівнює 0, та ані колір, ні зображення неможливо побачити.

Приклад

Клас clearfix

Клас clearfix був придуманий фронтенд-розробником Ніколасом Галахером (Nicolas Gallagher). Це, мабуть, найпопулярніший і найбільш ефективний на даний момент спосіб відміни обтікання. Полягає він в тому, що для псевдоелементів ::before та ::after селектора класу clearfix записуються наступні правила: 

.clearfix::before,
.clearfix::after {
    content: ""; 
    display: table; 
}
.clearfix::after { 
    clear: both; 
}

Потім сам клас записується в якості атрибуту для батьківського елементу: 

<div class="clearfix"> … </div>

В цьому випадку немає ані додаткових непотрібних елементів розмітки, ані приховування позиційованих елементів. Очищення обтікання назначається після останнього елементу всередині батьківського контейнера, так як описана в псевдоелементі ::after. Правила для ::before дозволяють уникнути проблем із схлопуванням верхніх відступів в сучасних браузерах.
