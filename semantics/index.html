Що таке семантичні елементи?

Семантичний елемент - елемент, який має сенс.

Семантичний елемент - це елемент, назва якого описує вміст елементу як для людини, так і для бравзера.

Приклади не семантичних елементів: <div> <span> - назва нічого не говорить про вміст елементу.

Приклади семантичних елементів: <img> <table> <article> - назва дає розуміння про вміст.

Багато веб-сайтів містять HTML-код на кшталт:

<div id="nav">
<div class="header">
<div id="footer">

для позначення навігації, заголовка та підвалу.

У HTML є кілька семантичних елементів, які можна використовувати для визначення різних частин веб-сторінки:

  • <article>
  • <aside>
  • <details>
  • <figcaption>
  • <figure>
  • <footer>
  • <header>
  • <main>
  • <mark>
  • <nav>
  • <section>
  • <summary>
  • <time>

Чому варто використовувати семантичні елементи?

  • Покращення читабельності коду – зрозуміліше для розробників, що означає кожен блок.
  • SEO (пошукова оптимізація) – пошукові системи краще розпізнають структуру сторінки.
  • Доступність – допомагають екранним читачам коректно інтерпретувати зміст.
  • Краща підтримка стандартів – HTML5 заохочує використання семантичних тегів.
  • Зменшення використання зайвих div – робить код більш логічним і організованим.

<section>

Елемент <section> визначає розділ у документі.

Розділ — це тематичне групування вмісту, зазвичай із заголовком.

Приклади використання елемента <section>:

  • глави (chapters)
  • вступ (introductions)
  • елементи нови (news items)
  • контактна інформація (contact info)

Зазвичай веб-сторінку можна розділити на розділи для вступу, вмісту та контактної інформації.

<article>

Елемент <article> визначає незалежний, самодостатній вміст.

Стаття повинна мати сенс самостійно, і її має бути можливо розповсюджувати незалежно від решти вебсайту.

Приклади використання елемента <article>:

  • дописи на форумах
  • дописи в блогах
  • коментарі користувачів
  • картки товарів
  • газетні статті

<header>

Елемент <header> представляє контейнер для вступного вмісту або набору навігаційних посилань.

Елемент <header> зазвичай містить:

  • один або більше заголовків (<h1> - <h6>)
  • логотип або іконку
  • інформацію про автора

Примітка: У HTML-документі може бути кілька елементів <header>. Однак <header> не можна розміщувати всередині <footer>, <address> або іншого <header>.

Приклад

<footer>

Елемент <footer> визначає нижній колонтитул документа або розділу.

Елемент <footer> зазвичай містить:

  • інформацію про автора
  • відомості про авторські права
  • контактну інформацію
  • карту сайту
  • посилання "на початок"
  • пов'язані документи

У документі може бути кілька елементів <footer>.

Приклад

<nav>

Елемент <nav> визначає набір навігаційних посилань.

Важливо зазначити, що не всі посилання в документі повинні бути всередині <nav>. Елемент <nav> призначений лише для основних блоків навігаційних посилань.

Браузери, такі як екранні рідери для людей з обмеженими можливостями, можуть використовувати цей елемент для визначення, чи слід спочатку пропустити його вміст при рендерінгу.

Приклад

<aside>

Елемент <aside> визначає деякий вміст, розташований окремо від основного вмісту (наприклад, бічну панель).

Вміст у <aside> повинен бути опосередковано пов'язаний із навколишнім контентом.
